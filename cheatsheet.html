Ceph 維運速查表

這個頁面是邊學邊整理的速查表，建議每週結束後回來補充對應的章節。以下先放好框架與第一週的內容，其餘章節隨進度填寫。

日常健康檢查指令

頻率指令檢查重點
每日ceph -s整體 health 是否為 HEALTH_OK，是否有 OSD down
每週ceph osd df各 OSD 容量使用率是否均衡，是否接近 nearfull
每週ceph osd treeOSD/host 拓撲是否與預期一致，是否有非預期的 down/out
異常時ceph health detail查看 HEALTH_WARN/ERR 的具體原因
異常時ceph pg dump列出所有 PG 狀態，找出非 active+clean 的 PG

PG 狀態碼對照表

狀態意義
active+clean正常，所有副本一致
active+degraded至少一個副本缺失，但仍可服務讀寫
peering正在重新確認副本一致性，短暫無法服務
recovering正在補齊少量缺失資料（pg log 仍有效）
backfilling正在大量搬移資料，影響較大
undersized副本數不足，CRUSH 找不到足夠位置
incomplete缺乏足夠資訊完成 peering，需人工介入（嚴重）

容量與 PG 數量

# 查看 autoscaler 建議
ceph osd pool autoscale-status

# 查看/調整 pool 的 pg_num、pgp_num
ceph osd pool get <pool> pg_num
ceph osd pool set <pool> pg_num <n>
ceph osd pool set <pool> pgp_num <n>

效能測試

# RADOS 層級基準測試
rados bench -p <pool> 60 write --no-cleanup
rados bench -p <pool> 60 seq
rados -p <pool> cleanup

# RBD 層級基準測試
rbd bench --io-type write --io-size 4K --io-threads 16 --io-total 1G <pool>/<image>

CRUSH map

# 匯出並轉成文字格式
ceph osd getcrushmap -o crushmap.bin
crushtool -d crushmap.bin -o crushmap.txt

監控告警對照表

項目建議等級備註
整體 health = HEALTH_ERR緊急立即通知
任一 OSD down/out緊急
OSD 使用率 > nearfull（約85%）警告
OSD 使用率 > full（約95%）緊急叢集會拒絕寫入
MON quorum 異常緊急
長時間 backfilling/recovering警告觀察是否影響效能

緊急排查步驟（草稿）

  1. ceph -s 確認整體狀態與警告類型
  2. ceph health detail 查看具體訊息
  3. 依訊息內容判斷是 OSD 問題、PG 問題還是 MON 問題，分別用 ceph osd tree / ceph pg dump / ceph mon stat 深入
  4. 記錄處理過程到實驗紀錄

提示：每完成一週的筆記，回來這頁補上對應的指令與心得，會比一次性整理更扎實。
